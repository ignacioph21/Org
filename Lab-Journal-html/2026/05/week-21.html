Estoy siguiendo las intrucciones de https://www.youtube.com/watch?v=IK25r7jOr4c para instalar Doom Emacs en Windows directamente.

  • Descargar todos los programas pertinentes.
    • Descargar Emacs del repositorio FTP: https://ftp.wayne.edu/gnu/emacs/windows/emacs-30/. Es el installer.exe de la versión 30.2 (la más reciente).
    • Descargar ripgrep del repo oficial: https://github.com/BurntSushi/ripgrep/releases. Para windows-gnu.
    • Descargar fd del repo oficial: https://github.com/sharkdp/fd/releases. Para windows-gnu.
    • Ya tenía descargado git de antes.
  • Intalar Emacs Vanilla con el instalador.
  • Extraer fd y ripgrep. Renombrarlos como fd y ripgrep y moverlos a Archivos de programa, que es donde se instaló Emacs.
  • Agregar una nueva environment variable llamada HOME que se dirija a C:/Users/Marcelo.
  • Agregar al Path:
    • C:\Program Files\Emacs\emacs-30.2\bin
    • C:\Program Files\fd
    • C:\Program Files\ripgrep
  • Correr en la consola git bash:
    • git clone https://github.com/hlissner/doom-emacs ~/.emacs.d
    • cd ~/.emacs.d/bin
    • ./doom install Tardó 1 hora y 13 segunda en instalarse.
    • ./doom sync Si todo funcionó correctamente.
  • Agregar al PATH C:\Users\Marcelo\.emacs.d\bin.
  • Quedaría instalar las fuentes con nerd-icons-install-fots y copiar el config.el y demás del WSL a Windows, usando symlinks para poder seguir usando el de WSL cuando sea necesario para editar el Journal.

Estuvimos haciendo varias pruebas y logramos hacer que rompa tipo spilling. Para esto bajamos la altura de agua del 1 cm que estaba sobre la parte plana a 0.35 cm. Además, usamos un impulso de 8 cm durante 1 s aproximadamente, con 15/20 s de espera después.

Lo que notamos es que si es demasiado rápido se vuelve un poco contraproducente porque empieza a salpicar agua y no se concentra tanto la onda. Además hay rebotes, incluso dejándolo descansar 20 segundos.

La idea ahora es probar con una rampa que suba suavemente al principio para poder concentrar más agua sin que llegue a salpicar.

[16:15]

Probando con una subida tipo tanh y con una caída tipo lineal en 15 s logramos ver mejor la rompiente tipo spilling. Juega mucho en que se forme:

  • La profundidad que está sumergida la paleta del motor. SI está muy poco sumergido salpica mucho y si está muy sumergido se genera una onda larga.
  • La distancia del motor a la rampa: Pareciera que cuanto más lejos, y más tiempo para formarse la onda, mejor se ve después.
  • La carrera del motor y el tiempo de esa carrera.
  • Ahora tiré el homing a 130 mm para poder dar carrera de hasta 10 cm en tiempos de 0.5 s.
  • SI llega a salpicar la impulsión se rompe la onda y por lo general manda varias más chicas que disminuyen el efecto de la rompiente.

A mejorar:

  • LA onda llega a la altura de la pared de PLA (y toca el perfil que lo sostiene).
  • Al tirar la playa muy para atrá el peso por flotación hace que pandee, habría que incorporar otro perfil de 2020 para sostenerlo en la punta.
  • Además la cuba está inclinada, entonces al tirarlo para atrá deja de estar todo sumergido. AGregamos agua de la canilla, aproximadamente 1/2 mm.

Además se llegan a formar si hay mucha agua o la impulsión es muy suave los Precursores de Sommerfeld, con mucha repetibilidad. Para cuantificar:

  • Hay que calcular el número de Iribarren y ver si corresponde con el tipo de rompiente que vemos.
  • Calcular el volumen de agua desplazado para saber la altura de la ola y planificar una pared más alta.
  • Probablemente hacer la zona plana del final más larga para colocar ahí el sensor, ya que rompen muy cerca de que esta comience.

Hay una diferencia notable entre la primer ola, generada con el agua calma, y lo que sucede luego cuando ya hay rebotes y demás. Una idea interesante sería poner un motor con paleta del otro lado, con este sistema tipo noise cancelling y un lazo de control, utilizando uno de los sesnores.
